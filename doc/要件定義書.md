# 要件定義書

**システム名**：見積書作成ツール

**作成日**：2026年7月

------------------------------------------------------------------------

# 1. システム概要

## 1.1 目的

業務計画書（PDF）と管理台帳（Excel）から必要な情報を抽出し、見積書フォーマットへ自動転記することで、見積書作成業務の効率化および転記ミスの削減を目的とする。

------------------------------------------------------------------------

## 1.2 背景

現在の見積書作成業務では、業務計画書や管理台帳から必要な情報を手作業で転記している。
これらを自動で情報を抽出・転記するツールを開発する。

------------------------------------------------------------------------

# 6. 機能要件

## 6.1 PDF選択機能

### 概要

業務計画書PDFを選択する。

### 入力

PDFファイル

### 出力

PDFパス

------------------------------------------------------------------------

## 6.2 管理台帳選択機能

### 概要

管理台帳Excelを選択する。

### 入力

Excelファイル

### 出力

Excelパス

------------------------------------------------------------------------

## 6.3 PDF解析機能

### 概要

業務計画書から必要項目を抽出する。

### 抽出項目

  項目
  ------------
  申請書No
  依頼部署
  件名
  成果物名称
  委託金額
  納期

------------------------------------------------------------------------

## 6.4 管理台帳検索機能

### 概要

PDFから取得した申請書Noを使用し、管理台帳を検索する。

### 検索方法

-   全シートを検索する
-   PDFの「ITK」を除去して検索する

例

    ITK14642
    ↓

    14642

### 取得項目

  列    取得内容
  ----- ----------
  D列   見積番号
  J列   発行日

------------------------------------------------------------------------

## 6.5 プレビュー機能

抽出した情報を画面へ表示する。

表示項目

-   申請書No
-   見積番号
-   発行日
-   依頼部署
-   件名
-   委託金額
-   納期
-   成果物名称

------------------------------------------------------------------------

## 6.6 Excel出力機能

### 処理概要

テンプレートExcelを開き、「フォーマット」シートのみを新規ブックへコピーし、見積書を作成する。

### 出力セル

  項目                   セル
  ---------------------- ---------
  見積番号               L1
  発行日                 K3
  依頼部署               A7
  件名                   E14
  成果物名称             B18以降
  委託金額               J18
  申請書No               C26
  納期                   C30
  納期（日付のみ置換）   J30

------------------------------------------------------------------------

## 6.7 出力時加工

### 依頼部署

末尾へ

    　御中

を付加する。

例

    第３シート設計部

    ↓

    第３シート設計部　御中

------------------------------------------------------------------------

### 委託金額

「円」を除去する。

例

    924,000円

    ↓

    924,000

------------------------------------------------------------------------

### 成果物名称

先頭の番号を削除する。

例

    ① H/R要件チェックまとめリスト

    ↓

    H/R要件チェックまとめリスト

------------------------------------------------------------------------

### 納期

J30セルの既存文章は残し、日付部分のみ置換する。

------------------------------------------------------------------------

### シート

-   シート名を「見積書」に変更する。
-   「見積書」以外のシートを削除する。

# 9. 非機能要件

-   Windows環境で動作すること
-   exe形式で配布できること
-   Microsoft Excelがインストールされていること
-   Excelの書式・結合セル・印刷設定を保持すること
-   入力ファイルを変更しないこと
-   テンプレートファイルを変更しないこと

------------------------------------------------------------------------

# 10. 制約事項

-   業務計画書のレイアウト変更時は解析処理の修正が必要となる。
-   管理台帳の列構成変更時は検索処理の修正が必要となる。
-   Microsoft
    Excelがインストールされていない環境ではExcel出力機能は利用できない。

# 12. Ver2.0追加仕様（Microsoft Graph・OneDrive対応）

## 12.1 管理台帳の管理方法変更

Ver2.0ではローカルのExcelファイル選択方式から、Microsoft Graph
APIを利用してOneDrive／SharePoint上の管理台帳を直接更新する方式へ変更した。

### 更新手順

1.  Microsoftアカウント認証（Device Code Flow）
2.  OneDrive／SharePointから管理台帳を一時ダウンロード
3.  管理台帳へデータを追記
4.  No・見積番号を自動採番
5.  発行日・発行者を設定
6.  OneDrive／SharePointへ再アップロード

## 12.2 Microsoft Graph認証

-   Microsoft Entra ID
-   Device Code Flow
-   アクセストークンキャッシュ
-   Microsoft Graph API利用

## 12.3 管理台帳更新

-   依頼部署から対象シート検索
-   C列から業務計画書No検索
-   最終行へ追加
-   No採番
-   見積番号採番
-   発行日・発行者記入

## 12.4 プレビュー追加項目

-   伝票番号
-   見積番号
-   発行日

## 12.5 Excel出力追加仕様

  ---------- ------
