# 1. システム概要

## 1.1 目的

本システムは、業務で使用するPDF・Excelファイルから必要な情報を取得し、
見積書の作成および管理台帳の更新を効率化することを目的とする。

また、OneDriveとの連携やAIによるPDF解析を利用し、
手作業による転記ミスや作業時間の削減を図る。

---

## 1.2 対象業務

本システムでは以下の4業務に対応する。

|業務|概要|
|:--|:--|
|特調TB|見積書作成・管理台帳更新|
|特調TB以外|見積書作成|
|TG|AI解析による見積書作成・管理台帳更新|
|MSR|見積書作成・管理台帳更新|

---

## 1.3 動作環境

|項目|内容|
|:--|:--|
|OS|Windows 11|
|Python|3.14|
|GUI|PySide6|
|Excel操作|openpyxl、Microsoft Excel|
|クラウド|OneDrive for Business|
|AI|OpenAI Vision|

# 3. 機能一覧

## 3.1 機能一覧

本システムが提供する機能を以下に示す。

|No.|機能名|概要|
|:--:|:--|:--|
|1|特調TB処理|PDFから情報を抽出し、見積書作成および管理台帳更新を行う。|
|2|特調TB以外処理|PDFから情報を抽出し、見積書を作成する。|
|3|TG処理|OpenAI Visionを利用してPDFを解析し、見積書作成および管理台帳更新を行う。|
|4|MSR処理|Excelファイルを読み込み、見積書作成および管理台帳更新を行う。|
|5|OneDrive連携|管理台帳の取得・更新を行う。|
|6|Excel出力|見積書をExcel形式で出力する。|
|7|PDF出力|必要に応じて見積書をPDF形式で出力する。|

---

## 3.2 業務フロー

各業務の基本的な処理の流れは共通である。

```text
入力ファイル選択
        │
        ▼
データ読込
        │
        ▼
情報抽出
        │
        ▼
見積書作成
        │
        ▼
管理台帳更新（対象業務のみ）

        ▼
処理完了
```

- OneDrive連携はCloudモジュールで実施する。
- AI解析はTG業務のみで利用する。

# 7. 外部連携

## 7.1 外部連携一覧

本システムでは、以下の外部サービスと連携する。

|連携先|用途|
|:--|:--|
|OneDrive for Business|管理台帳の取得・更新|
|Microsoft Graph API|OneDriveへの認証・ファイル操作|
|OpenAI Vision|TG業務におけるPDF解析|
|Microsoft Excel|見積書作成・PDF出力|

---

## 7.2 OneDrive連携

管理台帳はOneDrive上で管理し、処理時に最新ファイルを取得する。

処理完了後は更新した管理台帳をOneDriveへ保存する。

### 連携イメージ

```text
Estimate Tool
      │
      ▼
 Microsoft Graph API
      │
      ▼
 OneDrive
      │
      ├── 管理台帳取得
      └── 管理台帳更新
```

---

## 7.3 OpenAI連携

TG業務ではOpenAI Visionを利用してPDFを解析する。

解析結果をもとに見積書を作成する。

### 処理イメージ

```text
TG PDF
   │
   ▼
OpenAI Vision
   │
   ▼
抽出データ
   │
   ▼
見積書作成
```

---

## 7.4 Excel連携

Excelテンプレートを利用して見積書を作成する。

業務ごとに対応するテンプレートへデータを書き込み、Excelファイルとして保存する。

必要に応じてPDF形式でも出力する。

---

# 8. エラー処理

## 8.1 基本方針

利用者が原因を把握しやすいメッセージを表示し、安全に処理を終了する。

エラー発生時でもアプリケーションは終了せず、再実行できる設計とする。

---

## 8.2 エラー分類

|分類|内容|
|:--|:--|
|入力エラー|入力ファイル未選択、形式不正など|
|読込エラー|PDF・Excelの読込失敗|
|通信エラー|OneDrive・OpenAI接続失敗|
|保存エラー|見積書・管理台帳の保存失敗|
|システムエラー|予期しない例外|

---

## 8.3 エラー発生時の流れ

```text
エラー発生
     │
     ▼
処理中断
     │
     ▼
エラーダイアログ表示
     │
     ▼
利用者が内容を確認
```

---
