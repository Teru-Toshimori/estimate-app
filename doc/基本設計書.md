## 1. 文書情報

| 項目 | 内容 |
|---|---|
| システム名 | 見積書作成ツール |

---

# 2. システム概要

## 2.1 目的

本システムは、業務計画書PDFや見積書発行依頼Excelなどの入力ファイルから、見積書作成に必要な情報を取得し、見積書Excel・PDFの作成および管理台帳への登録を自動で行うWindowsデスクトップアプリケーションである。

利用者は、画面上部の共通入力欄に入力フォルダ、出力フォルダ、管理台帳URL、利用者一覧URLを設定し、対象業務のタブから「一括実行」ボタンを押す。

「一括実行」では、対象ファイルの検索、入力内容の読み取り、情報抽出、管理台帳更新、見積書作成、PDF出力および処理結果表示までを連続して実行する。

## 2.2 対象業務

| タブ | 対象業務 | 主な入力 |
|---|---|---|
| 特調TB | 特調TB案件の見積書作成 | 業務計画書PDF |
| 特調TB以外 | 特調TB以外案件の見積書作成 | 業務計画書PDF |
| TG | TG案件の見積書作成 | TG用PDF等 |
| MSR | MSR案件の見積書作成 | 見積書発行依頼Excel |

# 5. 機能設計

## 5.1 共通入力機能

### 入力フォルダ選択

- フォルダ選択ダイアログを表示する。
- 選択したフォルダパスを画面へ表示する。
- タブ切替時は、選択済みの共通入力を保持する。
- タブごとに必要な入力形式を検証する。

### 出力フォルダ選択

- Excel・PDFの保存先フォルダを選択する。
- 出力先が存在しない場合はエラーとする。
- 出力権限がない場合は処理を開始しない。

### URL貼り付け

- クリップボードの内容を対象欄へ設定する。
- 前後の空白文字を除去する。
- 空文字の場合は警告する。
- URL形式でない場合は実行前に警告する。

## 5.2 一括実行機能

```text
入力内容確認
  ↓
実行確認ダイアログ
  ↓
対象ファイル検索
  ↓
総件数表示
  ↓
ファイル単位の処理
  ├─ 入力ファイル読取
  ├─ 必要情報抽出
  ├─ 管理台帳取得
  ├─ 管理台帳検索・記入
  ├─ 見積／請求番号取得
  ├─ 見積書Excel作成
  ├─ 見積書PDF作成
  └─ 処理結果表示
  ↓
件数集計
  ↓
完了ダイアログ
```

個別のファイル選択、解析、台帳登録、Excel出力、PDF出力の各ボタンは設けず、「一括実行」に統合する。

## 5.3 処理中止機能

- 一括処理中のみ有効とする。
- ボタン押下時に中止確認を行う。
- 中止要求をWorkerへ通知する。
- 処理中のファイルは、安全に停止できる単位まで処理した後に終了する。
- 中止前に完了したファイルの出力内容は保持する。

## 5.4 Microsoft認証機能

Microsoft Graph API利用時は、Device Code Flowで認証する。

```text
一括実行
  ↓
アクセストークン確認
  ├─ 有効なトークンあり → 継続
  └─ トークンなし／期限切れ
       ↓
     デバイスログイン画面表示
       ↓
     利用者がMicrosoftへログイン
       ↓
     認証完了
```

## 5.5 OneDrive／SharePoint連携機能

- 管理台帳Excelのダウンロード
- 利用者一覧Excelのダウンロード
- 管理台帳Excelのアップロード
- ファイル情報およびETagの取得
- 競合検知
- ロック発生時のリトライ

## 5.6 利用者一覧取得機能

利用者一覧URLからExcelを取得し、担当者一覧を生成する。MSRタブでは、取得した担当者をコンボボックスへ表示する。

# 9. 見積書出力設計

## 9.1 共通仕様

- 業務別テンプレートを使用する。
- テンプレート原本は変更しない。
- 出力先に新しいExcelファイルを作成する。
- Excel作成後にPDFを出力する。
- 出力ファイル名は業務ごとの命名規則に従う。

# 10. エラー処理

## 10.1 入力エラー

| 条件 | 処理 |
|---|---|
| 入力フォルダ未選択 | 警告ダイアログ |
| 出力フォルダ未選択 | 警告ダイアログ |
| 管理台帳URL未入力 | 警告ダイアログ |
| 利用者一覧URL未入力 | 警告ダイアログ |
| MSR担当者未選択 | 警告ダイアログ |
| 対象ファイル0件 | 情報ダイアログ |
| 入力形式不正 | NGまたは失敗として一覧表示 |

## 10.2 外部連携エラー

| 条件 | 処理 |
|---|---|
| Microsoft認証失敗 | エラーダイアログ、処理中止 |
| OneDrive接続失敗 | エラーダイアログ、処理中止 |
| ファイル取得失敗 | 対象ファイルを失敗として表示 |
| 401／403 | 認証または権限エラー |
| 404 | URLまたは対象ファイル不明 |
| 412 | ETag競合として更新中止 |
| 423 | 一定回数リトライ後に失敗 |
| アップロード失敗 | 失敗として表示 |

## 10.3 ファイル処理エラー

| 条件 | 処理 |
|---|---|
| PDF解析失敗 | 対象ファイルを失敗として表示 |
| AI解析失敗 | TG対象ファイルを失敗として表示 |
| Excel読取失敗 | 対象ファイルを失敗として表示 |
| テンプレート不明 | 処理中止または対象ファイル失敗 |
| Excel出力失敗 | 対象ファイルを失敗として表示 |
| PDF出力失敗 | 対象ファイルを失敗として表示 |
| 管理台帳記入失敗 | 対象ファイルを失敗として表示 |

ファイル単位の失敗は、可能な限り次のファイルの処理を継続する。

# 11. 非機能設計

## 11.1 操作性

- 初めて使用する利用者でも、一括実行まで迷わない画面構成とする。
- 入力項目は画面上部へ集約する。
- タブ切替時に対象業務が明確に分かるタイトルを表示する。
- 処理中のファイル名と進捗率を表示する。
- 完了後に成功・NG・失敗件数を表示する。

## 11.2 性能

- 複数ファイルを連続処理できること。
- 時間のかかる処理はWorkerスレッドで実行すること。
- 画面操作が固まらないこと。
- 外部APIの応答待ちにはタイムアウトまたはリトライを設定すること。

## 11.3 セキュリティ

- APIキーや認証情報をソースコードへ記載しない。
- `.env` は公開リポジトリへコミットしない。
- Microsoft Graphは必要な権限のみ使用する。
- 認証トークンは外部へ表示しない。
- エラーログへAPIキーを出力しない。

## 11.4 保守性

- Reader、Writer、Cloud、AI、Excel処理を分離する。
- 業務別処理をタブおよびWorker単位で分離する。
- テンプレートパスはTemplateResolverで管理する。
- 共通Excel操作はExcelAutomationHelperへ集約する。
- メッセージ表示はCustomMessageDialogへ集約する。
